### Phase 1 implementation

Strategic-orchestrator pass executing `~/Documents/Projects/.plans/hydra/hydra-2026-05-13-tasks.md` Phase 1 via parallel subagents (Wave 1: Tasks 1.1+1.2+1.4 in parallel; Wave 2: 1.3+1.5 in parallel; Wave 3: 1.6). All six Phase 1 tasks complete with 100 tests green, ruff/format/vulture all clean.

- **Task 1.1** — `hydra/state.py` (WAL + busy-retry decorator with 5-attempt exponential backoff and 30-second circuit breaker), `hydra/migrations.py` (numbered, transactional migrations; split-and-execute preserves outer ROLLBACK), `hydra/audit.py` (single-writer JSONL mirror of state transitions, None-field-filtered).
- **Task 1.2** — `hydra/probe.py` multi-signal live-session probe: no `session_end` AND (audio.wav OR audio_seg*.wav OR transcript.jsonl mtime within 5 minutes). - **Task 1.4** — `hydra/subprocess_runner.py` process-group-aware async spawner (`process_group=0`), in-memory PGID registry, SIGINT/SIGTERM/atexit cleanup walking `killpg(pgid, SIGTERM)` → 2s grace → `SIGKILL`. Test verifies child lives in its own pgroup distinct from Hydra's, so cleanup cannot take down the parent.
- **Task 1.5** — `tests/conftest.py` autouse recording-integrity fixture. Patches `builtins.open`/`Path.write_text`/`Path.write_bytes`/`os.fsync`. Caller-frame discrimination via `sys._getframe` walk: writes are policed only when the calling stack contains a frame under `<repo>/hydra/`; pytest, sqlite, watchdog, site-packages all pass through. Opt-out marker `@pytest.mark.allow_writes_anywhere` registered in `pyproject.toml`. Fork+exec limitation documented (ai_monitor subprocess writes correctly invisible by construction — PM-2).
- **Task 1.6** — `hydra/__main__.py` argparse routing for `start`/`status`/`stop`/`report`/`finalize`/`prune`. `start` resolves the session via probe → initializes state.db → runs `TranscriptTailer` printing events to stdout (Phase 1 smoke test). `--wait-for-session SECS` and `--background` flags wired; background path forks AFTER probe so the user sees attach confirmation synchronously before the parent returns. ### Phase 2 implementation

- **Task 2.1** — `hydra/models.py` central model registry. `MODEL_TIERS` dict over `watcher` / `fast` / `heavy` tiers; `ModelSpec` dataclass (`cli`, `model`, `effort_flag`, `hard_timeout_s`); `default_tiers()` returns a deep copy; `load_model_tiers(config_path)` merges `[defaults.models]` overrides from config.toml (watcher tier accepts `primary`/`fallback` shorthand; fast/heavy tiers accept arrays of full ModelSpec tables). Watcher default flipped from `local-gemma` to `claude:claude-haiku-4-5` per pre-mortem PM-1 (zero GPU contention with Scarecrow's Parakeet); `config.example.toml` updated with the new default and a documented override template.
- **Task 2.2** — `hydra/watcher.py` rolling-window topic flagger. Maintains a deque of transcript events within `window_seconds` (default 30); ticks every `tick_seconds` (default 15); builds prompt → invokes `model_invoker(prompt, current_model)` → parses JSON `[{topic, confidence, rationale}]`. Auto-fire at confidence ≥ 0.7, suggest at ≥ 0.4, ignore below. Dedup uses char-Jaccard ≥ 0.5 OR substring overlap against a 5-minute rolling list of recent topics. Failure surfacing: yellow "watcher unstable" banner on the FIRST failure (immediate, no silent failures); flip to fallback after 2 failures in a rolling 30s window per PM-3. Already-on-fallback + further failures emit error banners but don't re-switch. Successful tick resets failure tracking. The watcher injects its `model_invoker`, `on_flag`, `on_banner`, `next_q_id`, and `clock` so tests are deterministic; production wires them to subprocess_runner, state.insert_question, audit.emit, and time.monotonic.
- **Task 2.3** — `scripts/perf_test_local_watcher.py` skeleton landed (CLI surface, report-writing, pass/fail computation, p95 statistics helper). **Actual perf-test run is deferred to manual hardware verification** because it requires Scarecrow's Parakeet pipeline running concurrently with mlx-vlm Gemma on the user's machine. Script gates execution behind `--i-have-read-the-docstring` so it cannot run accidentally in CI. Until the manual run is completed and PASS is documented here, the watcher default remains cloud Haiku.

### Phase 3 implementation

- **Task 3.1** — `hydra/quota.py` quota router + `hydra/ai_monitor_schema.py` Pydantic v2 schema. The schema models ai_monitor's *actual* `--json --once` output (read from `~/Documents/Projects/ai_monitor/ai_monitor/{ui.py,providers.py,parsing.py}`), not the suggested-but-wrong shape from the design brief: each provider entry is `{name, ok, source, data, display, error}` and the remaining-percentage key lives inside provider-specific `data` (Claude → `session_percent_left`/`weekly_percent_left`/`opus_percent_left`; Codex → `five_hour_percent_left`/`weekly_percent_left`; Gemini → `flash_percent_left`/`pro_percent_left`; Vibe → `usage_percent` which is inverted, % used not remaining). Provider names in ai_monitor are capitalized (`Claude`); Hydra uses lowercase (`claude`) so a `by_provider_lower()` accessor bridges the gap. Failed/errored providers return `0.0` remaining so they never win selection. Router behavior: 60s in-memory cache, per-provider 60s blacklist for mid-flight 429s, round-robin fallback when ai_monitor errors or returns an unparseable schema, `quota_low` event when every candidate in a tier is below 5%. `NoCandidateModelError` raised when all providers in a tier are blacklisted or unavailable. Events for `schema_mismatch` and `ai_monitor_unavailable` fire only once per router lifetime.
- **Task 3.2** — `hydra/cli_check.py` pre-flight availability gate. `shutil.which` for PATH detection (no hardcoded `/opt/homebrew`); per-tool `--version` invocation with a 5-second timeout via injectable `run` hook; structured `CheckResult` (`name`, `present`, `path`, `version`, `error`, `install_hint`). Version parsing prefers stdout's first non-empty line and falls back to stderr (some CLIs print `--version` to stderr). `available_set()` returns the names whose result is `present=True AND error is None`, wiring directly into `QuotaRouter(cli_available=...)` so missing-tool tier members are excluded from the routing pool. `preflight_check()` raises `AllToolsMissingError` if not a single configured tool is usable; otherwise returns the full per-tool dict for the pre-flight UI to render.

Test inventory grew by 33: quota router (18), CLI availability (15) = **173 passing in 11.55s**.

### Phase 4 implementation

- **Task 4.1** — `hydra/json_extract.py` brace-depth scanner that pulls JSON out of noisy CLI stdout. Pre-strips ` ```json ` / ` ```javascript ` / ` ``` ` fences via regex, then runs a single-pass char scanner tracking `in_string`/`escape` state and a matched-type stack so `}` cannot close `[`. Invalid slices fall forward to the next opener (recovery for "garbage {not json} actually {\"a\": 1}"-style inputs). `extract_all` returns every value in document order for tool-call-noise outputs that contain multiple JSON blobs. Seven static fixtures committed at `tests/fixtures/model_outputs/` covering clean/with-preamble/with-fences/with-suffix/with-tool-call-noise/deeply-nested cases (SR-4 nested-brace fixture verifies the embedded `{` inside `"snippet": "She said: \"go { left } at the fork\""` does NOT terminate the outer object).
- **Task 4.1** — `hydra/citations.py` per-claim citation validator. `ValidatedAnswer` (answer prose + Citations list + unsourced_claims list + warnings). `validate()` rejects structurally-broken responses (`CitationValidationError`); per-citation it requires `id`/`source_type`/`quoted_snippet`, web → `url`, local → `file_path`. Duplicate IDs keep first occurrence; orphan `[N]` references are stripped from the answer with a warning; unsourced-typed citations' snippets accumulate into `unsourced_claims` while keeping the inline reference in the answer.
- **Task 4.2** — `hydra/worker.py` (`run_research_job`) + `hydra/dispatcher.py` (two-queue coordinator). Worker flow: pick model via `QuotaRouter.pick_model(tier)` → build prompt (meeting context + question + citation contract + JSON schema) → spawn via `subprocess_runner.spawn` with `process_group=0` and `hard_timeout_s` from the model spec → on timeout, terminate the process group (SIGTERM → 200ms → SIGKILL) and for `heavy` tier auto-retry once with a different model (achieved by mark-then-pick on the router) → extract JSON → validate citations → write `q-NNN.md` to `<session>/hydra/research/` → update `state.set_question_status` and emit `job_succeeded`. Mid-flight 429 detection (regex over `\b429\b` / `rate[_\s-]?limit` / `quota exceeded`) blacklists the provider and retries up to 3 distinct providers before failing the job. "All-claims-unsourced" surfaces as a soft flag on `job.error` while still writing the artifact (worker checks `validated.unsourced_claims AND no web/local citations`). Dispatcher pulls flags via `enqueue_flag(flag)` (which inserts the question row into state.db first), enqueues both a `fast` and a `heavy` job for the same q_id, and runs `quick_concurrency` (default 3) + `deep_concurrency` (default 2) workers off two independent `asyncio.Queue`s. Quick failures don't cancel deep jobs for the same q_id; `stop()` cancels worker tasks cleanly.
- **Post-dispatch fix-up** — the implementer's `_finalize_failure` initially fired the event via `asyncio.ensure_future` (fire-and-forget) so the test harness saw an empty event list. Stepping in directly: made the helper `async`, awaited the emit, fixed the all-claims-unsourced detector to compare against "no web/local citations" rather than "no citations at all" (which broke when the model returned a single `source_type="unsourced"` citation), and tidied a handful of lint nits (UP041 `(TimeoutError, asyncio.TimeoutError)` → bare `TimeoutError` since Python 3.11+ aliases them; SIM105 `try/except/pass` → `contextlib.suppress`; E501 line-wrap on the CitationValidationError message; unused noqa removals).

Test inventory grew by 77: JSON extraction (24), citation validation (22), worker subprocess (22), dispatcher flows (9) = **250 passing in 12.13s**.

### Phase 5 implementation

- **Task 5.1** — `hydra/indexer.py` BM25 corpus indexer over SQLite FTS5. Default DB at `~/Documents/Projects/hydra/.cache/index.sqlite` (allowlisted in conftest). `init_db()` creates the FTS5 virtual table (`files(corpus_root UNINDEXED, path UNINDEXED, content, tokenize='unicode61')`) and a `file_meta(corpus_root, path, mtime, size, indexed_at)` companion table for incremental caching keyed by `(corpus_root, path)`. `index(corpus_roots)` runs each root as its own `asyncio` task; file I/O dispatches via `asyncio.to_thread`. Skip rules: `.git/` `node_modules/` `.venv/` `__pycache__/` `.obsidian/` `.DS_Store` (plus defensive additions for `.pytest_cache/` `.ruff_cache/` `.mypy_cache/`); `.DS_Store` filename; files >5MB; binary mimetype via `mimetypes.guess_type`; broken symlinks. `os.walk(followlinks=False)` plus in-place `dirnames[:]` mutation prunes the walk before the OS stats anything inside skipped directories. FTS5 inserts use delete-then-insert (no clean UPSERT on virtual tables); the companion meta table uses real `ON CONFLICT ... DO UPDATE`. Files removed from disk are pruned from both tables on the next index pass. Progress events emit every 20 files; status events emit at "running" / "ready" / "error" transitions. `query(q, limit=20)` runs an FTS5 `MATCH` with SQLite's built-in `bm25()` ranking.
- **Thread-safety note**: the indexer DB connection uses `check_same_thread=False` plus a `threading.Lock` because `asyncio.to_thread` dispatches to a worker pool whose threads aren't pinned. Only DB writes serialize on the lock; file reads and walks parallelize.

Test inventory grew by 24: indexer happy-path (6), indexer incremental (5), indexer skip rules (13) = **274 passing in 12.20s**.
